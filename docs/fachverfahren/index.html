Fachverfahren-Strategie

Wie 300+ Fachverfahren Ende-zu-Ende an den VerwDigiStack angebunden werden – und warum Hersteller intrinsische Motivation entwickeln, Stack-konform zu werden.

1 · Der Engpass: Fachverfahren

Der eigentliche Engpass der Verwaltungsdigitalisierung
300+ verschiedene Fachverfahren · Veraltete Client-Server-Technik · Keine APIs · Vendor Lock-in · Schnittstellen als Profit-Center der Hersteller · Behörden können nicht wechseln.

Der Medienbruch: Heute vs. Zielbild

Heute (Medienbruch)
  • Online-Formular → E-Mail/PDF
  • Sachbearbeiter:in druckt aus
  • Tippt in Fachverfahren ein
  • Bescheid wird gedruckt
  • Brief per Post → Bürger:in
  • Dauer: 2–6 Wochen
Zielbild (Ende-zu-Ende)
  • Deutschland-App → API-Gateway
  • BPM-Engine → Fachverfahren-API
  • Automatische Prüfung + Bearbeitung
  • Bescheid (QES-signiert) → API
  • Push-Notification → Bürger:in
  • Dauer: Minuten bis Tage

Warum „einfach APIs vorschreiben" scheitert

Grund Erklärung
Kein Marktdruck Behörden können Anbieter nicht wechseln → Hersteller haben keinen Anreiz
Investitionsschutz Millionen in bestehende Systeme investiert; Wechselkosten prohibitiv
Fehlende Standards Ohne klare Specs weiß niemand, was „API-fähig" heißt
Regulierungslücke Keine Rechtsgrundlage für API-Pflicht
Profit-Center Hersteller verdienen 50.000–500.000 EUR pro individueller Anbindung

2 · Stack-Konformitätsmodell

Kernidee
Stack-Konformität wird zum Wettbewerbsvorteil, nicht zur Pflicht. Das Ökosystem erzeugt den Druck: Nur Stack-konforme Produkte können am Deutschland-Portal und der Deutschland-App teilnehmen.

Die drei Konformitätsstufen

Stack-Connected
Übergangslösung für Legacy
  • Adapter-Layer vor dem Fachverfahren
  • 7 Pflicht-API via Adapter-SDK
  • Screen-Scraping / RPA / DB-Bridge
  • XÖV → REST-Translator
  • Rückmeldung ggf. verzögert
Stack-Native
Fachverfahren implementiert APIs direkt
  • OpenAPI 3.1 Endpunkte nativ
  • OAuth 2.0 / mTLS
  • CloudEvents für Status-Updates
  • Consent-Token-Validierung
  • Echtzeit-Status für Bürger:innen
Stack-Cloud-Native
SaaS, Container, API-First
  • Multi-Tenant SaaS
  • Containerized (K8s)
  • Auto-Scaling
  • SLA 99,9%
  • Optimale UX + Proaktive Features

4 · Ökosystem-Design: Die 7 Hebel

Hebel 1: Stack-Marktplatz
Pull-Anreiz
„Wer Stack-konform ist, erreicht alle 11.000 Behörden."
Zentraler Marktplatz für zertifizierte Fachverfahren. Behörden können erstmals vergleichen: Bewertungen, Preise, Konformitätsstufe. Hersteller sehen den Anreiz: Ein Produkt, 11.000 potenzielle Kunden. Wettbewerb wird sichtbar.
Hebel 2: Vergaberecht-Kopplung
Push-Anreiz
„Ab 2028: Nur Stack-konforme Fachverfahren beschaffbar."
Stack-Konformität wird MUSS-Anforderung im Vergaberecht. Bestandsschutz für laufende Verträge; Nachfolge muss Stack-konform sein. Migrationszuschuss: 50% der Kosten für erste 3 Jahre. Übergangsfrist: 3 Jahre Stufe 1, 5 Jahre Stufe 2.
Hebel 3: Conformance-Test-Suite
Enabler
„Konformität prüfen in 5 Minuten – automatisiert."
Open-Source-CLI (stack-test run --level 2). Testet alle 7+4 APIs automatisch. CI/CD-Integration (GitHub Actions, GitLab CI). Sandbox-Umgebung mit Mock-APIs aller BBs. Kostenlos für Hersteller.
Hebel 4: Stack-Connect-Adapter-SDK
Enabler
„Legacy anbinden in Tagen, nicht Monaten."
Fertiger Open-Source-Adapter (Apache 2.0) für Bestandssysteme. Konfigurieren statt Programmieren: YAML-Mappings, SQL-Queries, Filesystem-Watch, OSCI-Bridge. Betrieb als Docker-Container.
Hebel 5: Wirtschaftliches Ökosystem
Pull-Anreiz
„SaaS-Marge: 70–80% vs. On-Prem: 50–60%."
Neues Geschäftsmodell: Pay-per-Transaction statt Lizenz. Multi-Tenant Cloud steigert Marge. Marktplatz-Reichweite ersetzt Vendor Lock-in. Weniger Support durch standardisierte APIs. Neue Player (Start-ups) können einsteigen.
Hebel 6: Zertifizierungsprogramm
Signal
„Zertifizierung = Vertrauen + Sichtbarkeit."
Drei Stufen: Connected (kostenlos), Native (5.000 EUR), Cloud-Native (15.000 EUR). Automatisierter Test + Pen-Test + UX-Review. Gültig 2 Jahre; jährliches Re-Testing. Zertifizierte Produkte erhalten Premium-Positionierung im Marktplatz.
Hebel 7: Open-Source-Referenzimplementierungen
Druck + Enabler
„Behörden haben immer eine Open-Source-Alternative."
Für die Top-20-Domänen: vollständig funktionsfähige Open-Source-Fachverfahren. Lizenz EUPL/Apache 2.0. Direkt einsetzbar oder als Template. Senkt Eintrittsbarriere für neue Anbieter. Erzeugt Wettbewerbsdruck auf proprietäre Hersteller.

Geschäftsmodell-Transformation

Altes Modell
  • Lizenz + Wartung + individuelle Anbindungen
  • Vendor Lock-in sichert Umsatz
  • Jede Behörde einzeln verhandeln
  • API-Integration als Zusatzgeschäft
  • On-Premises pro Behörde
  • Wechselkosten = Burggraben
Neues Modell (Stack-Ökosystem)
  • SaaS + Pay-per-Transaction + Marktplatz
  • Marktplatz-Reichweite sichert Umsatz
  • Einmal zertifizieren → 11.000 Behörden
  • API ist Standard → Kernprodukt ist besser
  • Multi-Tenant Cloud → Margensteigerung
  • UX-Qualität = Burggraben
